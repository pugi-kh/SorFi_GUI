기존에 CLI로 작업했던 MerFi를 GUI로 만들면서 이름도 함께 변경.
Merge보다 Assort가 더 의미적으로 맞는것 같음.

python과 tkinter를 활용
대략적인 인터페이스는 "사용자 시나리오"를 참고하여 Mockup파일을 만듬.
Mockup을 참고하여 인터페이스 구현

SorFi_main.py 파일에서 인터페이스를 구현하고 메인이 되는 기능은 SorFi_run에서 구현

인터페이스
    - 작업할 폴더 경로
    - 메인 옵션
    - 생성할 폴더명 리스트
    - 문자열 위치 확인 여부

메인기능
    - 파일 옮기기 실행
    - 직전실행 되돌리기
        : 현재 변수에 저장된 값들을 활용하여 
          "실행경로+폴더명"의 경로에서 전체 파일을 "실행경로" 위치로 이동한 후,
          "폴더명" 리스트의 폴더들을 삭제.
          --> 생각해보니 기존에 폴더가 있었거나, 프로그램 실행의 결과가 아닌 사용자가 임의로 파일을 옮겼을 경우가 있을 듯
              "실행경로"를 설정하고 실행할 때, 해당 경로 내에 파일명을 리스트화하여 임시 저장.
              해당 리스트 내의 파일만 되돌리기 실행.
              되돌리기 실행할 때, 파일명이 해당 리스트에 있는지 확인 후 실행.
    - 현재 작성한 설정을 저장 (JSON 활용 예정)
    ex)
    {   폴더 경로 : C:\, 
        폴더명 : { abc, def, ghi, jkl, mno, pqr, stu }, 
        문자열 위치 확인 여부 : True,
        시작위치 : 3
    }
    - 설정 불러오기 (JSON 불러오기)


현재 인터페이스까지는 어느정도 작성이 완료된 상태.
추가적인 내용은 메인기능을 구현하면서 추가하면 될 듯.
